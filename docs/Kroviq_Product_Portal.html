Open Source

Kroviq Core

The automation engine. A reusable Java framework consumed as a Maven dependency by any test project.

  • Selenium 4 WebDriver utilities
  • Cucumber BDD step definitions
  • Kroviq Engine — 6 UI framework table engines
  • Multi-framework wrappers (AntD, AG Grid, MUI, Angular Material, PrimeNG)
  • Enterprise components (FileUpload, DatePicker, MultiSelect, Tree, Form, Toast)
  • REST API testing with schema validation
  • Windows Desktop automation via WinAppDriver
  • Dynamic test data (JSON + Excel)
  • Self-healing locators + smart waits
  • Extent + custom HTML/Excel reporting

Automatic root cause analysis on test failures. Rule-based engine classifies failures by category (locator, timing, assertion, environment, driver) and assigns ownership.

Structure

Project Layout

Clean separation of concerns across packages.

Kroviq_Core/
├── src/main/java/kroviq/
│   ├── ai/                    # AI capabilities (RCA, Defect Writer, Spark)
│   ├── api/                   # REST API testing (ApiClient, Auth, Schema)
│   ├── constants/             # Framework-wide constants
│   ├── hooks/                 # Cucumber lifecycle hooks
│   ├── model/                 # Data models (FlexiField, FlexiFieldType)
│   ├── parallel/              # Parallel execution runner
│   ├── reporting/             # Extent, Excel, HTML report generators
│   ├── utils/                 # Core utilities (WaitHandler, SmartLocator, etc.)
│   └── wrapper/               # Multi-framework UI wrappers
│       ├── aggrid/            # AG Grid engine + handler
│       ├── antd/              # Ant Design handlers
│       ├── angularmaterial/   # Angular Material handlers
│       ├── core/              # Interfaces (TableEngine, DatePicker, etc.)
│       ├── desktop/           # Windows Desktop (WinAppDriver)
│       ├── factory/           # ComponentHandlerFactory + Generic handlers
│       ├── mui/               # Material UI handlers
│       └── primeng/           # PrimeNG handlers
├── src/test/                  # 400+ framework unit/integration tests
├── docs/                      # API freeze docs, release notes, guides
└── pom.xml                    # Maven build (com.kroviq:kroviq-core:2.0.0)

Yes. Kroviq supports thread-safe parallel execution via TestContext isolation. Configure thread count in RunManager.json or execution profiles.

AI features are rule-based and run offline — no external API calls. RCA uses pattern matching on exceptions, Defect Writer templates from context, and Spark uses a 3-stage pipeline with a step catalog.

Kroviq Engine handles table-level operations (find row, sort, filter, paginate). Wrappers handle individual component interactions (dropdowns, date pickers, file uploads). Both auto-detect the UI framework.
